Plugin Development

Create and publish plugins to extend Cocapn agent capabilities

The npm of AI Agents: Cocapn plugins are discoverable, installable, and versioned extensions that anyone can publish. Think of them as npm packages for AI agents.

What are Plugins?

Plugins are published skill cartridges that extend Cocapn agent capabilities. A plugin bundles one or more skills along with metadata, tests, optional runtime modules, and personality fragments.

Why Plugins Matter

The plugin system creates a flywheel effect:

  • Users get instant access to specialized capabilities (GitHub, Slack, data analysis)
  • Developers reach thousands of Cocapn users without building their own agent runtime
  • Ecosystem grows as useful plugins attract more users, which attracts more developers

Plugin Package Format

Directory Structure

my-plugin/
├── cocapn-plugin.json      # Plugin manifest (required)
├── README.md               # User-facing documentation (required)
├── LICENSE                 # License file (recommended)
├── skills/
│   ├── my-skill.ts          # Skill implementation
│   ├── my-skill.test.ts     # Skill tests (required)
│   └── helpers/
│       └── github-client.ts # Helper modules
├── modules/                 # Optional runtime modules
│   ├── brain/               # Brain modules
│   ├── ui/                  # UI components
│   ├── cloud/               # Cloud modules
│   └── tools/               # MCP tool definitions
├── personality.md           # Optional personality fragment
└── assets/                  # Static assets (icons, etc.)
    └── icon.png

  "$schema": "cocapn-plugin-schema-v1",

  "author": "Superinstance <team@superinstance.com>",

  "homepage": "https://github.com/superinstance/cocapn-plugin-github#readme",
  "bugs": "https://github.com/superinstance/cocapn-plugin-github/issues",
  "keywords": ["github", "issues", "pull-requests", "code-review"],
  "category": "development",
  "icon": "assets/icon.png",

  "skills": [
    {
      "name": "github-issues",
      "entry": "skills/github-issues.ts",
      "type": "hot",
      "tolerance": {
        "maxTokens": 2000,
        "timeout": 30000
      },
      "triggers": ["issue", "bug", "pr", "pull request", "github"],
      "description": "List, create, and manage GitHub issues"
    }
  ],

  "dependencies": {
    "cocapn-core": ">=0.1.0"
  },

  "permissions": [
    "network:api.github.com",

    "node": ">=18.0.0",

  }
}

Manifest Fields

Field Required Description
name Yes Plugin name (must start with cocapn-plugin-)
version Yes Semver version string
description Yes Short description (max 200 chars)
author Yes Author name or email
license Yes SPDX license identifier
skills Yes Array of skill definitions
permissions Yes Required permissions
repository No Git repository URL
keywords No Search keywords (max 10)

Permission Model

Plugins declare required permissions in their manifest. Users approve permissions during installation.

Permission Types

Permission Description Example
network:<host> Network access to specific host network:api.github.com
network:* Dangerous Unrestricted network access —
fs:read:<path> Read filesystem under path fs:read:~/repos
fs:write:<path> Write filesystem under path fs:write:~/output
shell:<cmd> Execute specific shell command shell:gh, shell:git
env:<var> Read environment variable env:GITHUB_TOKEN

Permission Flow

  1. User installs plugin
  2. CLI shows required permissions
  3. User approves (or denies specific permissions)
  4. Approved permissions stored in plugin state
  5. Runtime enforces sandbox
Security: Permissions are enforced at runtime. Hot skills run in the bridge process with permission checks. Cold skills run in isolated subprocesses with permissions as capability flags.

Example Skill (TypeScript)

import { Octokit } from "octokit";

export interface SkillContext {
  brain: Brain;
  logger: Logger;
  signal: AbortSignal;
}

export default async function githubIssues(
  query: string,
  context: SkillContext
): Promise<string> {
  const token = process.env.GITHUB_TOKEN;
  if (!token) {
    throw new Error("GITHUB_TOKEN not set");
  }

  const octokit = new Octokit({ auth: token });

  // Parse query: "issues in superinstance/cocapn"
  const match = query.match(/issues\s+in\s+(\S+)/);
  if (!match) {
    return "Usage: issues in <repo> (e.g., 'issues in superinstance/cocapn')";
  }

  const [, repo] = match;
  const [owner, name] = repo.split("/");

  const { data: issues } = await octokit.rest.issues.listForRepo({
    owner,
    repo: name,
    state: "open",
    per_page: 10,
  });

  if (issues.length === 0) {
    return `No open issues in ${repo}`;
  }

  return `Open issues in ${repo}:\n${issues.map(i =>
    `• #${i.number}: ${i.title} (${i.user?.login})`
  ).join("\n")}`;
}

Skill Entry Points

Skills must export a default function that takes a query and context:

export default async function mySkill(
  query: string,
  context: SkillContext
): Promise<string> {
  // Your skill logic here
  return "Result";
}

The SkillContext provides access to:

  • brain — Agent memory (facts, wiki, tasks)
  • logger — Structured logging
  • signal — AbortSignal for cancellation

Publishing Plugins

Step 1: Validate

cocapn plugin validate

Step 2: Test

cocapn plugin verify cocapn-plugin-github
# → Running tests in sandbox...
# → ✓ 12/12 tests passed

Step 3: Publish

cocapn plugin publish
# → Validating cocapn-plugin.json...
# → Running tests...
# → Building artifact...
# → Uploading to registry...
# → Published cocapn-plugin-github@1.2.0
Automatic Scans: On publish, the registry runs security scans for known vulnerabilities, license compatibility, and suspicious code patterns.

CLI Commands

Discovery

# Search plugins
cocapn plugin search github

# Browse by category
cocapn plugin search --category development

# Get plugin details
cocapn plugin info cocapn-plugin-github

# List installed plugins
cocapn plugin list

Installation

# Install latest version
cocapn plugin install cocapn-plugin-github

# Install specific version
cocapn plugin install cocapn-plugin-github@1.1.0

Management

# Uninstall a plugin
cocapn plugin uninstall cocapn-plugin-github

# Update all plugins
cocapn plugin update

# Update specific plugin
cocapn plugin update cocapn-plugin-github

# Check for updates
cocapn plugin outdated

Permissions

# Revoke specific permission
cocapn plugin revoke-permission cocapn-plugin-github network:api.github.com

# Grant additional permission
cocapn plugin grant-permission cocapn-plugin-github shell:git

# View plugin permissions
cocapn plugin permissions cocapn-plugin-github

Example Plugins

cocapn-plugin-github

GitHub integration — issues, PRs, code review. Features:

  • List, create, and manage GitHub issues
  • Analyze and review pull requests
  • Permissions: network:api.github.com, shell:gh

cocapn-plugin-slack

Slack messaging — send messages, channels, users. Features:

  • Send messages to Slack channels or users
  • Permissions: network:api.slack.com, env:SLACK_TOKEN

cocapn-plugin-data-analysis

Data analysis — CSV, JSON, statistics. Features:

  • Analyze CSV files — stats, histograms, correlations
  • Query JSON files using JSONPath
  • Permissions: fs:read, shell:python3

Registry API

The Cocapn plugin registry provides:

  • Search: GET /api/plugins/search?q=github&category=development
  • Details: GET /api/plugins/:name
  • Publish: POST /api/plugins/publish
  • Versions: GET /api/plugins/:name/versions

Built with Cloudflare Workers, AdmiralDO, R2, and D1.
