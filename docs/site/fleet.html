Fleet Protocol

Multi-agent coordination across devices and domains

What is a Fleet?

A fleet is a group of Cocapn bridge instances that trust each other. Fleet members can:

  • Route A2A messages — Send messages between domains
  • Share fleet keys — No GitHub PAT sharing needed
  • Sync via Git — Stay in sync via shared private repo or cross-repo Git refs
Use Cases: Personal multi-device (desktop + laptop + cloud), multi-domain (makerlog ↔ studylog), team collaboration (multiple people's bridges working together).

Fleet Topologies

Same private repo, multiple machines:

Fleet Authentication

Fleet Keys

Fleet members authenticate with short-lived JWTs signed by a shared fleet key:

Header: { "alg": "HS256", "typ": "JWT" }
Payload: {
  "sub": "bridge-<instance-id>",
  "iss": "cocapn",
  "iat": <unix-seconds>,
  "exp": <iat + 3600>,
  "dom": "makerlog.ai"   // optional: domain claim
}
Signature: HMAC-SHA256(header + payload, fleet-key)

Tokens are valid for 1 hour. The bridge auto-refreshes them before expiry.

Fleet Key Storage

The fleet key is stored in secrets/fleet-key.age (age-encrypted):

  • Generated on first bridge init
  • Synced via Git to all fleet members
  • Decrypted with the bridge's identity key
  • Never transmitted over the network

Setting Up a Fleet

Personal Fleet (Same Repo)

Machine 1 (Primary)

# Initialize the bridge — this generates the fleet key
cocapn-bridge --repo ~/brain

# The fleet key is generated on first run
ls ~/brain/secrets/fleet-key.age

Machine 2 (Secondary)

# Pull the private repo (fleet-key.age is already in it)
git clone git@github.com:you/your-brain.git ~/brain

# Restore your age identity key
# Option A: copy ~/.config/cocapn/identity.age from Machine 1
# Option B: export from OS keychain

# Start the bridge — it loads the fleet key automatically
cocapn-bridge --repo ~/brain

Both bridges now accept fleet JWTs signed by the shared key.

Cross-Domain Fleet (Different Repos)

Step 1: Generate Fleet Keys

# On bridge A (makerlog.ai)
cocapn-bridge secret init

# On bridge B (studylog.ai)
cocapn-bridge secret init

Step 2: Exchange Public Keys

# On bridge A
cocapn-bridge fleet show-public-key
# → fleet-pubkey-makerlog.ai

# On bridge B
cocapn-bridge fleet add-trusted-key makerlog.ai <public-key>

Step 3: DNS Verification

# Add DNS record on studylog.ai
_cocapn.studylog.ai CNAME fleet.cocapn.io

Step 4: Configure Trusted Domains

# cocapn/config.yml — fleet section
fleet:
  acceptedSuffixes:
    - fleet.cocapn.io
    - fleet.yourdomain.com
  trustedDomains:
    - studylog.ai
    - activelog.ai

A2A Messaging

Message Format

Agent-to-Agent (A2A) messages use JSON over HTTP POST:

{
  "type": "A2A",
  "targetDomain": "studylog.ai",
  "agentId": "default",
  "content": "What were my study goals for this week?",
  "replyTo": "makerlog.ai",
  "messageId": "msg-123",
  "timestamp": "2026-03-29T12:00:00Z"
}

Routing Flow

  1. Local bridge receives A2A request
  2. Verifies targetDomain is in trustedDomains
  3. Checks DNS CNAME for _cocapn.<domain>
  4. Signs request with fleet JWT
  5. POSTs to remote bridge's A2A endpoint
  6. Streams response back to requester

Example Request

# From makerlog.ai bridge
curl -X POST https://studylog.ai/api/a2a \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <fleet-jwt>" \
  -d '{
    "type": "A2A",
    "targetDomain": "studylog.ai",
    "agentId": "default",
    "content": "What were my study goals for this week?",
    "replyTo": "makerlog.ai"
  }'

Response

{
  "type": "A2A_RESPONSE",
  "content": "Your study goals for this week:...",
  "agentId": "default",
  "domain": "studylog.ai",
  "messageId": "msg-123",
  "timestamp": "2026-03-29T12:00:01Z"
}

Fleet Configuration

config.yml Fleet Section

# cocapn/config.yml
fleet:
  # Accepted CNAME suffixes for DNS verification
  acceptedSuffixes:
    - fleet.cocapn.io
    - fleet.yourdomain.com

  # Trusted domains (can send A2A messages)
  trustedDomains:
    - studylog.ai
    - activelog.ai
    - makerlog.ai

  # Fleet message timeout (seconds)
  messageTimeout: 30

  # Enable fleet logging
  logMessages: true

Environment Variables

# Fleet key path (default: secrets/fleet-key.age)
COCAPN_FLEET_KEY_PATH=/custom/path/fleet-key.age

# Fleet JWT TTL (default: 3600 seconds)
COCAPN_FLEET_JWT_TTL=7200

Cloudflare Admiral (Optional)

When cocapn/cocapn-cloud.yml is present with an admiralUrl, all bridges in the fleet register their heartbeats with the Admiral Durable Object.

Admiral Features

  • Heartbeat tracking — See which bridges are online
  • Automatic routing — Route to nearest available bridge
  • Session continuity — Resume conversations on different machines
  • Task queue — Distributed task execution

Configuration

# cocapn/cocapn-cloud.yml
cloudflare:
  admiralUrl: https://admiral.yourdomain.workers.dev
  apiToken: "secret:CLOUDFLARE_API_TOKEN"

  # Heartbeat interval (seconds)
  heartbeatInterval: 30

  # Enable automatic routing
  autoRouting: true

Security Considerations

Fleet Key Security

  • Never commit fleet keys to public repos
  • Use age encryption (default) for fleet key storage
  • Rotate fleet keys periodically
  • Revoke compromised keys immediately

DNS Verification

  • Always verify DNS CNAME before routing
  • Use acceptedSuffixes to limit allowed CNAMEs
  • Monitor for unauthorized DNS changes

Message Validation

  • Verify JWT signature on every A2A message
  • Check message timestamp (prevent replay attacks)
  • Validate targetDomain is in trustedDomains
