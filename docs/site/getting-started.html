Getting Started

Deploy your first Cocapn AI instance in 60 seconds

Prerequisites

Before you begin, ensure you have the following:

  • Node.js 18+ — Cocapn runs on Node.js
  • Git — For version control and Git-backed memory
  • Cloudflare account — For Workers deployment (free tier works)
  • GitHub account — For hosting your brain repos (private + public)
Note: The Cloudflare Workers free tier includes 100,000 requests per day. This is sufficient for personal use and small projects.

Installation

Option 1: Global CLI (Recommended)

Install the Cocapn CLI globally:

npm install -g cocapn

Option 2: Create Cocapn (Scaffold)

Create a new Cocapn instance without global installation:

npx create-cocapn my-instance

Option 3: Development Mode

If you want to contribute or run from source:

git clone https://github.com/CedarBeach2019/cocapn.git
cd cocapn/packages/local-bridge
npm install
npm link

Create Your Instance

Step 1: Initialize

Create a new Cocapn instance:

cocapn init my-instance
cd my-instance

This creates:

  • cocapn.yml — Public configuration (agents, modules, templates)
  • cocapn-private.yml — Private configuration (secrets, API keys)
  • soul.md — Agent personality and system prompt
  • brain/ — Private Git repo for agent memory
  • public/ — Public Git repo for published content

Step 2: Configure

Edit cocapn.yml to customize your instance:

# Public configuration
domain: my-instance.ai
title: "My AI Instance"
description: "A personal AI assistant"

agents:
  default:
    model: claude-sonnet-4
    temperature: 0.7
    maxTokens: 4096

modules:
  - name: habit-tracker
    type: brain
  - name: perplexity-search
    type: tool

Step 3: Initialize Private Brain

Create your private brain repository:

# Create private repo on GitHub first, then:
cd brain
git init
git remote add origin git@github.com:your-username/your-brain.git
cocapn brain init  # Sets up memory structure
git add .
git commit -m "Initial brain"
git push -u origin main
Security: Your private brain repo contains encrypted secrets. Never commit cocapn-private.yml to a public repository.

Deploy to Cloudflare Workers

Step 1: Authenticate with Cloudflare

cocapn cloudflare:login

Step 2: Deploy

cocapn deploy

This command:

  1. Bundles your instance for Workers
  2. Creates a Cloudflare Worker
  3. Deploys to your-instance.workers.dev
  4. Sets up environment variables and secrets
  5. Prints your Workers URL

Step 3: Verify

Visit your Workers URL to verify deployment:

https://your-instance.subdomain.workers.dev
Success! Your AI instance is now live on Cloudflare Workers. It's accessible 24/7, even when your local machine is offline.

Run Locally

For development or when you don't need cloud deployment:

cocapn start

This starts the local bridge server on ws://localhost:8787.

Connect a Client

Use the React UI or any WebSocket client:

# From another terminal
cd packages/ui
npm install
npm run dev

Visit http://localhost:5173 to interact with your agent.

Custom Domain

To use a custom domain with Cloudflare Workers:

  1. Add your domain to Cloudflare
  2. Update cocapn.yml:
domain: my-instance.ai
customDomain: true
  1. Deploy again:
cocapn deploy --domain my-instance.ai
Note: Custom domains require a Cloudflare account with the domain added. The free Cloudflare tier works great for this.

Troubleshooting

Deployment Fails

Ensure you're authenticated with Cloudflare:

cocapn cloudflare:login
cocaln cloudflare:whoami

Bridge Won't Start

Check that port 8787 is available:

lsof -i :8787  # macOS/Linux
netstat -ano | findstr :8787  # Windows

Agent Not Responding

Check bridge logs:

cocapn logs

For more troubleshooting tips, see Troubleshooting.md.
