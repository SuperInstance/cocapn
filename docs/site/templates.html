Template Guide

Themed AI deployments for every use case

What are Templates?

Cocapn templates are curated starting points for AI deployments. Each template includes:

  • Personality — A unique soul.md that defines the agent's character
  • Configuration — Pre-configured cocapn.yml with agents and modules
  • Memory Structure — Initial facts, wiki pages, and tasks
  • Skills — Domain-specific capabilities pre-installed
  • UI Theme — Custom styling and layout
All features are installable on any domain. Templates are just curated starting points. You can mix and match features across templates.

Template Anatomy

Directory Structure

templates/my-template/
├── public/                 # Public template assets (Vite app)
│   ├── index.html
│   ├── main.tsx
│   └── style.css
├── private/                # Private template assets
│   ├── soul.md             # Agent personality
│   ├── config.yml          # Cocapn configuration
│   ├── facts.json          # Initial facts
│   ├── wiki/               # Initial wiki pages
│   │   └── welcome.md
│   └── procedures/         # Initial procedures
│       └── onboarding.md
├── modules/                # Bundled modules
│   └── custom-module/
└── README.md               # Template documentation

soul.md — Agent Personality

The soul.md file defines the agent's personality, system prompt, and behavior:

# You are a Dungeon Master AI

You are an expert TTRPG dungeon master with decades of experience running
engaging campaigns across multiple systems (D&D 5e, Pathfinder, Call of Cthulhu).

## Your Expertise

- World-building and narrative design
- Rule systems and mechanics
- Improvisation and player engagement
- Character development and storytelling

## Your Style

- Descriptive and immersive
- Fair and consistent
- Player-focused (not adversarial)
- Flexible (adapt to player choices)

## Boundaries

- Never make decisions for players
- Always explain rules when asked
- Keep the game fun and engaging
- Respect the table's tone preferences

config.yml — Cocapn Configuration

# Template: DMlog
domain: dmlog.ai
title: "DMlog AI — Your Dungeon Master Assistant"
description: "AI-powered TTRPG campaign management"

agents:
  default:
    model: claude-sonnet-4
    temperature: 0.8
    maxTokens: 4096
    systemPromptFile: soul.md

modules:
  - name: character-sheet
    type: brain
  - name: dice-roller
    type: tool
  - name: campaign-manager
    type: brain

skills:
  - name: create-npc
    entry: skills/create-npc.ts
    type: hot
    triggers: ["npc", "character", "create character"]
    description: "Create detailed NPCs for your campaign"

Using Templates

Create from Template

# Use a built-in template
npx create-cocapn my-campaign --template dmlog

# Use a custom template
npx create-cocapn my-instance --template /path/to/template

Apply Template to Existing Instance

cocapn template apply dmlog

List Available Templates

cocapn template list
# → personallog    — Generic personal assistant
# → makerlog       — Developers & manufacturers
# → dmlog          — TTRPG campaigns
# → studylog       — Education & research

Creating Custom Templates

Step 1: Initialize Template

mkdir my-template
cd my-template
cocapn template init

Step 2: Configure Personality

Edit private/soul.md to define your agent's personality:

# You are a specialized AI assistant

Your purpose is to [describe purpose].

## Your Expertise

- [List key areas]
- [List key areas]

## Your Style

- [Describe tone and approach]
- [Describe interaction style]

Step 3: Configure Skills

Add skills to private/config.yml:

skills:
  - name: my-custom-skill
    entry: skills/my-custom-skill.ts
    type: hot
    triggers: ["trigger", "keyword"]
    description: "What this skill does"

Step 4: Add Initial Memory

Create initial facts and wiki pages:

# private/facts.json
{
  "user.name": "Your Name",
  "user.goal": "Template goal",
  "template.version": "1.0.0"
}

Step 5: Test Template

cocapn template validate
cocapn template test

Step 6: Publish Template

cocapn template publish
# → Validates template structure
# → Bundles template assets
# → Publishes to registry
# → Template URL: https://templates.cocapn.ai/my-template

Template Customization

Override Personality

After creating from a template, you can customize the personality:

# Edit soul.md
vim ~/my-instance/cocapn/soul.md

# Restart bridge to apply changes
cocapn restart

Add Modules

cocapn module add habit-tracker
cocapn module add perplexity-search

Configure Domain

Edit cocapn.yml to use a custom domain:

domain: my-custom-domain.ai
customDomain: true

Template Publishing

Template Registry

Publish templates to the Cocapn template registry for others to discover:

cocapn template publish --public
# → Validates template
# → Runs tests
# → Publishes to registry
# → https://templates.cocapn.ai/username/my-template

Template Metadata

Create a template.json file in your template:

{
  "name": "my-template",
  "version": "1.0.0",
  "description": "A specialized AI template for...",
  "author": "Your Name <you@example.com>",
  "license": "MIT",
  "repository": "https://github.com/you/my-template",
  "keywords": ["ai", "assistant", "specialized"],
  "category": "productivity",
  "icon": "assets/icon.png"
}

Template Best Practices

1. Start Simple

Begin with the personallog template and customize incrementally.

2. Document Everything

Include a comprehensive README.md explaining your template's purpose and usage.

3. Test Thoroughly

Use cocapn template test to validate your template before publishing.

4. Version Control

Track template versions in template.json and follow semantic versioning.

5. Respect User Privacy

Never include sensitive data or secrets in public templates.
