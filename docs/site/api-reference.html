API Reference

Complete API documentation for Cocapn

WebSocket Protocol

Cocapn uses WebSocket for real-time communication between clients and the bridge. All messages are JSON.

Connection

ws://localhost:8787  # Local bridge
wss://your-instance.workers.dev  # Cloudflare Workers

Authentication

Authenticate by sending an AUTH message as the first message:

{
  "type": "AUTH",
  "token": "github_pat_...",  // GitHub PAT or fleet JWT
  "version": "1.0.0"
}

Message Types

Type Direction Description
AUTH Client → Server Authenticate with GitHub PAT or fleet JWT
AUTH_SUCCESS Server → Client Authentication successful
AUTH_ERROR Server → Client Authentication failed
CHAT Client → Server Send chat message to agent
STREAM_CHUNK Server → Client Stream chunk from agent
STREAM_END Server → Client Agent finished streaming
BASH Client → Server Execute shell command
BASH_OUTPUT Server → Client Shell command output
FILE_EDIT Client → Server Edit a file
FILE_EDIT_RESULT Server → Client File edit result
RPC Client → Server Remote procedure call
RPC_RESULT Server → Client RPC result
MODULE_INSTALL Client → Server Install a module
MODULE_INSTALL_RESULT Server → Client Module install result

WebSocket Methods

CHAT

Send a chat message to an agent:

{
  "type": "CHAT",
  "agentId": "default",
  "content": "Hello, agent!",
  "stream": true,
  "temperature": 0.7,
  "maxTokens": 4096
}

Response (Streaming)

// Stream chunks
{
  "type": "STREAM_CHUNK",
  "agentId": "default",
  "content": "Hello",
  "done": false
}

// Stream end
{
  "type": "STREAM_END",
  "agentId": "default",
  "tokensUsed": 42
}

BASH

Execute a shell command:

{
  "type": "BASH",
  "command": "ls -la",
  "cwd": "/home/user/projects"
}

Response

{
  "type": "BASH_OUTPUT",
  "stdout": "total 8...",
  "stderr": "",
  "exitCode": 0
}

FILE_EDIT

Edit a file:

{
  "type": "FILE_EDIT",
  "path": "/path/to/file.txt",
  "oldText": "old text",
  "newText": "new text"
}

Response

{
  "type": "FILE_EDIT_RESULT",
  "success": true,
  "path": "/path/to/file.txt"
}

RPC

Remote procedure call:

{
  "type": "RPC",
  "method": "brain.queryFacts",
  "params": {
    "query": "user.name"
  }
}

Response

{
  "type": "RPC_RESULT",
  "result": "John Doe",
  "error": null
}

MODULE_INSTALL

Install a module:

{
  "type": "MODULE_INSTALL",
  "name": "habit-tracker",
  "type": "brain",
  "url": "https://github.com/cocapn-modules/habit-tracker"
}

Response

{
  "type": "MODULE_INSTALL_RESULT",
  "success": true,
  "module": {
    "name": "habit-tracker",
    "type": "brain",
    "version": "1.0.0"
  }
}

REST API (Cloud Worker)

When deployed to Cloudflare Workers, Cocapn exposes a REST API for HTTP clients.

Base URL

https://your-instance.workers.dev/api

Authentication

Include a Bearer token in the Authorization header:

Authorization: Bearer github_pat_...

Endpoints

POST /api/chat

Send a chat message (non-streaming):

POST /api/chat
Content-Type: application/json

{
  "agentId": "default",
  "content": "Hello, agent!",
  "temperature": 0.7,
  "maxTokens": 4096
}

Response

{
  "content": "Hello! How can I help you?",
  "tokensUsed": 42,
  "agentId": "default"
}

POST /api/brain/facts

Query or update facts:

POST /api/brain/facts
Content-Type: application/json

{
  "action": "query",
  "query": "user.*"
}

Response

{
  "facts": {
    "user.name": "John Doe",
    "user.email": "john@example.com"
  }
}

GET /api/brain/wiki

List wiki pages:

GET /api/brain/wiki

Response

{
  "pages": [
    {
      "title": "Welcome",
      "path": "wiki/welcome.md",
      "updatedAt": "2026-03-29T12:00:00Z"
    }
  ]
}

GET /api/brain/wiki/:page

Get wiki page content:

GET /api/brain/wiki/welcome.md

Response

{
  "title": "Welcome",
  "content": "# Welcome\n\nThis is your wiki...",
  "path": "wiki/welcome.md"
}

POST /api/brain/tasks

Manage tasks:

POST /api/brain/tasks
Content-Type: application/json

{
  "action": "create",
  "task": {
    "content": "Build documentation site",
    "tags": ["docs", "priority"]
  }
}

Response

{
  "task": {
    "id": "task-123",
    "content": "Build documentation site",
    "status": "pending",
    "tags": ["docs", "priority"],
    "createdAt": "2026-03-29T12:00:00Z"
  }
}

GET /api/status

Get bridge status:

GET /api/status

Response

{
  "status": "online",
  "version": "1.0.0",
  "uptime": 3600,
  "agents": ["default", "coder"],
  "modules": ["habit-tracker", "perplexity-search"]
}

GET /api/health

Health check endpoint:

GET /api/health

Response

{
  "healthy": true,
  "timestamp": "2026-03-29T12:00:00Z"
}

MCP Tools

Cocapn exposes agent capabilities through the Model Context Protocol (MCP).

Available Tools

mcp_brain_query_facts

Query the agent's fact database:

{
  "name": "mcp_brain_query_facts",
  "arguments": {
    "query": "user.name"
  }
}

mcp_brain_set_fact

Set a fact in the agent's memory:

{
  "name": "mcp_brain_set_fact",
  "arguments": {
    "key": "user.preference",
    "value": "dark-mode"
  }
}

mcp_brain_search_wiki

Search the agent's wiki:

{
  "name": "mcp_brain_search_wiki",
  "arguments": {
    "query": "project setup"
  }
}

mcp_git_commit

Commit changes to Git:

{
  "name": "mcp_git_commit",
  "arguments": {
    "message": "Update documentation"
  }
}

mcp_git_push

Push changes to remote:

{
  "name": "mcp_git_push",
  "arguments": {}
}

mcp_shell_exec

Execute a shell command:

{
  "name": "mcp_shell_exec",
  "arguments": {
    "command": "npm test",
    "cwd": "/path/to/project"
  }
}
